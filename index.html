About

I am a BCA student with a deliberate focus on backend development and structured programming. My approach is to understand systems before building them — breaking problems into components, designing clean interfaces, and writing modular code.

I work primarily in Python, building CLI applications that practice real design principles: CRUD operations, JSON-based persistence, input validation, and function-level separation of concerns.

Currently strengthening core fundamentals before expanding into full backend frameworks. The goal is a solid base in data flow, system architecture, and database integration — built to last, not just to work.
